This post demonstrates the three ways to add images.

## 1. Plain Markdown (from /public)

The simplest way — drop a file in `public/blog/` and reference it with a relative URL:

![A Moleskine notebook open to a ruled page](/blog/example-inline.jpg)

The file lives at `public/blog/example-inline.jpg`. URL is `/blog/example-inline.jpg`.
No optimization, no resizing — you ship whatever you put there.

## 2. The cover image (set in frontmatter)

The image at the top of this post — and on the journal index card — came from the
`cover:` field in this file's frontmatter. One field, two places.

## 3. Optimized `<Image>` mid-post (this is .mdx)

Astro's `<Image>` component generates WebP + AVIF variants and computes the
aspect ratio, so the page doesn't jump on load:

<Image src={diagram} alt="The four-column note: context, constraint, success, gratitude" width={900} />

Imported from `src/assets/blog/four-columns.png`. Astro processes it at build time.

## Tips

- Cover images: 1200×630px ships cleanly to social previews.
- Inline images: max 1600px wide, target ~200KB JPGs.
- Use the `coverAlt:` field for screen-readers — it doubles as the alt text in OG cards.
