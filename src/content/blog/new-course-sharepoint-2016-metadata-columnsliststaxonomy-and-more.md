## Common SharePoint mistakes the course explicitly fixes

**Building taxonomy bottom-up instead of top-down.** Teams start by listing their existing folders and turning each one into a managed term. Result: an enormous flat term store nobody can navigate. The course teaches the *top-down* method — start with the half-dozen biggest organizational concepts (department, document type, lifecycle stage, audience) and build down.

**Treating SharePoint like OneDrive with permissions.** Document libraries with no metadata become drag-and-drop dumping grounds. The fix isn't more training; it's *making the metadata mandatory at upload time* via content types. Three required fields beats a six-page upload guide that nobody reads.

**One taxonomy per site.** Each new SharePoint site gets its own term store and its own field definitions. Six months later there are 12 sites with 12 ways to mean "client name." The course shows how to define managed metadata once at the tenant level and reuse everywhere.

**Skipping the search refiner config.** SharePoint's search is *actually good* — but only if you've told it which managed properties to refine on. Default config = useless search. The course walks the configuration in module five.

**Not auditing for orphan content types.** Five years in, you'll have content types no document uses anymore plus content types used by 50 docs that nobody knows are duplicates. Quarterly audit. The course gives you the script.

## The 90-minute taxonomy workshop the course teaches you to run

The first module ships a *workshop facilitator's guide* — a 90-minute session you can run with any team to draft a working taxonomy before lunch. The shape: 15 minutes of "what do we file" inventory, 30 minutes of "group these into 5-7 buckets" affinity-mapping, 20 minutes of "name the buckets with terms a new hire would understand," 15 minutes of validation against your three biggest use cases, 10 minutes of wrap-up and ownership assignment.

The workshop's secret weapon is the **"new-hire test"** — could a person who joined the company yesterday find a document using only the taxonomy terms, no tribal knowledge? If no, the terms aren't ready. *That single test eliminates 90% of jargon-based taxonomies.*
